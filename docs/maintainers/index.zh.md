## 维护者职责

### 代码审查

- 在 48 小时内审查所有传入的 PR
- 确保代码质量和标准
- 提供建设性反馈
- 合并已批准的 PR

### 社区管理

- 监控 Telegram 和 GitHub 讨论
- 回答社区问题
- 分类问题和功能请求
- 认可贡献者

### 发布管理

- 规划发布计划
- 创建发布说明
- 标记和发布版本
- 更新文档

## 访问要求

维护者需要访问：

- GitHub 仓库（写权限）
- Telegram 管理员群组
- 发布管理工具

联系核心团队请求访问权限。
