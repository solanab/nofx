# Community

Welcome to the NOFX community! This section contains everything you need to contribute and participate.

## How to Contribute

### Getting Started

1. **Read the Guides**
   - [Contributing Guide](https://github.com/tinkle-community/nofx/blob/main/CONTRIBUTING.md) - Complete contribution workflow
   - [Code of Conduct](https://github.com/tinkle-community/nofx/blob/main/CODE_OF_CONDUCT.md) - Community standards
   - [Security Policy](https://github.com/tinkle-community/nofx/blob/main/SECURITY.md) - Report vulnerabilities

2. **Find Something to Work On**
   - Browse [GitHub Issues](https://github.com/tinkle-community/nofx/issues)
   - Look for `good first issue` label
   - Check out [bounty tasks](#bounty-program)

3. **Join the Community**
   - :speech_balloon: [Telegram Developer Community](https://t.me/nofx_dev_community)
   - :bird: [Twitter @nofx_official](https://x.com/nofx_official)
   - :octopus: [GitHub Discussions](https://github.com/tinkle-community/nofx/discussions)

## Bounty Program

### Active Bounties

NOFX offers bounties for valuable contributions:

| Category | Reward Range | Examples |
|----------|--------------|----------|
| :1st_place_medal: Major Features | $500-1000 | Exchange integration, core architecture |
| :2nd_place_medal: Medium Features | $200-500 | WebSocket support, new AI models |
| :3rd_place_medal: Small Features | $50-200 | Bug fixes, UI improvements, documentation |

### How to Claim Bounties

[Complete Bounty Guide](bounty-guide.md){ .md-button .md-button--primary }

**Quick Steps:**

1. Find issue tagged `[BOUNTY]`
2. Comment with your proposal
3. Wait for approval
4. Work on the task
5. Submit PR with demo
6. Get paid after merge!

## Recognition

### Contributor Levels

- :star: **Active Contributor** - Submit quality PRs
- :star2: **Trusted Contributor** - 3+ merged PRs, given review rights
- :gem: **Core Team** - Top contributors, invited by maintainers

### Benefits

- Listed in README and release notes
- Direct access to maintainer discussions
- Priority support for your issues
- Invitation to private roadmap planning

## Contribution Types

### Code Contributions

- New exchange integrations
- AI model adapters
- Bug fixes and improvements
- Performance optimizations

**Required:**

- :white_check_mark: Code compiles and runs
- :white_check_mark: Follows code style guidelines
- :white_check_mark: Includes basic tests (preferred)
- :white_check_mark: Updates documentation if needed

### Documentation

- Tutorial writing
- Translation (Chinese, Russian, Ukrainian, etc.)
- FAQ updates
- Video guides

### Testing & QA

- Bug reports with reproduction steps
- Security vulnerability reports
- Testnet verification
- Performance testing

## Community Guidelines

### Our Values

- **Respect** - Treat everyone with courtesy
- **Transparency** - Open communication and decisions
- **Quality** - High standards for contributions
- **Collaboration** - Work together, help each other

### Not Acceptable

- :x: Harassment or discrimination
- :x: Spam or self-promotion

- :x: Violating Code of Conduct
