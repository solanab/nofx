# 社区

欢迎来到 NOFX 社区！本节包含参与和贡献所需的一切。

---

## 如何贡献

### 开始

1. **阅读指南**
   - [贡献指南](https://github.com/tinkle-community/nofx/blob/main/CONTRIBUTING.md) - 完整的贡献工作流程
   - [行为准则](https://github.com/tinkle-community/nofx/blob/main/CODE_OF_CONDUCT.md) - 社区标准
   - [安全政策](https://github.com/tinkle-community/nofx/blob/main/SECURITY.md) - 报告漏洞

2. **找到要做的事**
   - 浏览 [GitHub Issues](https://github.com/tinkle-community/nofx/issues)
   - 寻找 `good first issue` 标签
   - 查看 [悬赏任务](#悬赏计划)

3. **加入社区**
   - :speech_balloon: [Telegram 开发者社区](https://t.me/nofx_dev_community)
   - :bird: [Twitter @nofx_official](https://x.com/nofx_official)
   - :octopus: [GitHub Discussions](https://github.com/tinkle-community/nofx/discussions)

## 悬赏计划

### 活跃悬赏

NOFX 为有价值的贡献提供悬赏：

| 类别 | 奖励范围 | 示例 |
|------|----------|------|
| :1st_place_medal: 重大功能 | $500-1000 | 交易所集成、核心架构 |
| :2nd_place_medal: 中等功能 | $200-500 | WebSocket 支持、新 AI 模型 |
| :3rd_place_medal: 小功能 | $50-200 | Bug 修复、UI 改进、文档 |

### 如何领取悬赏

[完整悬赏指南](bounty-guide.md){ .md-button .md-button--primary }

**快速步骤:**

1. 找到标记 `[BOUNTY]` 的 issue
2. 评论你的方案
3. 等待批准
4. 开始工作
5. 提交 PR 和演示
6. 合并后获得报酬！

## 认可

### 贡献者级别

- :star: **活跃贡献者** - 提交高质量 PR
- :star2: **受信任贡献者** - 3+ 合并的 PR，获得审查权限
- :gem: **核心团队** - 顶级贡献者，由维护者邀请

### 福利

- 在 README 和发布说明中列出
- 直接访问维护者讨论
- 你的 issue 优先支持
- 邀请参加私人路线图规划

## 贡献类型

### 代码贡献

- 新交易所集成
- AI 模型适配器
- Bug 修复和改进
- 性能优化

**要求:**

- :white_check_mark: 代码可以编译和运行
- :white_check_mark: 遵循代码风格指南
- :white_check_mark: 包含基本测试（首选）
- :white_check_mark: 如需要则更新文档

### 文档

- 编写教程
- 翻译（中文、俄语、乌克兰语等）
- FAQ 更新
- 视频指南

### 测试和 QA

- 带有复现步骤的 Bug 报告
- 安全漏洞报告
- 测试网验证
- 性能测试

## 社区准则

### 我们的价值观

- **尊重** - 礼貌对待每个人
- **透明** - 开放的沟通和决策
- **质量** - 高标准的贡献
- **协作** - 共同努力，互相帮助

### 不可接受的行为

- :x: 骚扰或歧视
- :x: 垃圾信息或自我推销
- :x: 分享恶意代码
- :x: 违反行为准则
